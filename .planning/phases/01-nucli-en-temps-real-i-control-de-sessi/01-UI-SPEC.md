# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope of this contract:** the two foundational surfaces of CodeArena — the **Admin panel** (team registration, live monitoring grid, global timer controls) and the **Team screen** (dedicated waiting state, inter-phase interstitial, and the split action-panel/preview layout shell). Game content (drag & drop, CSS holes, JS rules, scoring) is out of scope for Phase 1 but this contract sets the tokens and phase color-coding that Phases 2–5 MUST reuse (UX-02).

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Vanilla JS + Vite — React/shadcn explicitly rejected per `.claude/CLAUDE.md`) |
| Preset | not applicable |
| Component library | none — hand-rolled components over CSS custom properties (design tokens) |
| Icon library | Lucide (per-icon SVG import via Vite, no icon font, no JS runtime) |
| Font | Inter — self-hosted/bundled via Vite (`@fontsource/inter` or local woff2), NOT CDN; `font-display: swap` |

**Token strategy:** all values below are declared as CSS custom properties on `:root` in a single `tokens.css` imported by both `admin.html` and `client.html`. Executor MUST reference tokens (`var(--space-md)`, `var(--color-accent)`, `var(--phase-html)`), never hard-coded literals in component styles. This is the single source of visual truth shared across all 5 phases.

**shadcn initialization gate:** skipped — confirmed. Stack is Vanilla JS + Vite, not React; shadcn requires JSX. Registry safety gate is not applicable.

## Spacing Scale

Declared values (multiples of 4). Exposed as `--space-*` custom properties.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline padding inside chips/badges |
| sm | 8px | Compact spacing (button internal padding, status dot gap) |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section padding, gap between team cards in the grid |
| xl | 32px | Layout gaps, admin panel outer margins |
| 2xl | 48px | Major section breaks (registration block ↔ monitoring grid) |
| 3xl | 64px | Page-level spacing, vertical centering offsets on waiting screen |

Exceptions:
- **Touch/click target minimum:** all interactive controls (buttons, team-selection list items, timer control icons) render at a minimum 44px hit area even when visually smaller, to reduce misclicks under classroom time pressure.
- **Split layout:** the Team screen action-panel/preview split uses a percentage/`fr` grid (see Layout Contract), not the spacing scale, for the column division itself; the scale still governs padding inside each pane.

## Typography

Font family: **Inter**, weights 400 (regular) and 600 (semibold) only — bundled locally via Vite, `font-display: swap`.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.4 |
| Heading | 24px | 600 | 1.2 |
| Display (timer) | 56px | 600 | 1.0 |

Notes:
- **Timer digits** use `font-variant-numeric: tabular-nums` so the countdown does not jitter as digits change width. Displayed on the Team screen and Admin panel, synchronized (CORE-04).
- Exactly 4 sizes, exactly 2 weights — no additional variants permitted this phase.
- Waiting-screen team name renders at Heading size (24px/600); the "Connectat ✓, esperant el professor" secondary line renders at Body.

## Color

Base theme: **LIGHT**.

### 60 / 30 / 10 surface split

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F8FAFC` | App background, page canvas (both Admin and Team) |
| Secondary (30%) | `#FFFFFF` | Cards (team cards, registration block), the action panel and preview pane surfaces; borders `#E2E8F0` |
| Accent (10%) | `#059669` (emerald) | Primary CTA only — the "Iniciar Fase" button and other primary go-actions |
| Destructive | `#DC2626` (red) | Destructive/confirming actions only (Resync confirmation) |

Text colors: primary text `#0F172A` (slate-900), secondary/muted text `#64748B` (slate-500).

**Accent reserved for:** the single primary action button per screen (Admin: "Iniciar Fase" / "Següent fase"). NOT for secondary admin controls (Pausar/Reprendre, +1 minut, Resync, Registrar), which use a neutral secondary button style (white surface, slate border, slate-900 text). NEVER "all interactive elements".

### Semantic status colors (NOT part of the 60/30/10 accent budget)

Small status indicators only (dots/icons on team cards), never large surfaces:

| Status | Value | Usage |
|--------|-------|-------|
| Connected | `#16A34A` | Team card connection dot/icon = connected (ADMIN-05) |
| Disconnected | `#94A3B8` | Team card connection dot/icon = disconnected (grayed, calm — not alarming red) |

### Phase color-coding (UX-02 — cross-phase contract, reused in Phases 2–5)

Language-logo colors, reused identically HTML→CSS→JS on every screen (interstitial accent, phase badge, active-phase indicator). Exposed as `--phase-html`, `--phase-css`, `--phase-js`.

| Phase | Value | Usage |
|-------|-------|-------|
| HTML | `#E44D26` (orange) | HTML-phase badge, interstitial accent, phase indicator |
| CSS | `#264DE4` (blue) | CSS-phase badge, interstitial accent, phase indicator |
| JS | `#D4B106` (dark yellow) | JS-phase badge, interstitial accent, phase indicator — darkened from logo `#F0DB4F` for AA contrast on light bg |

### Timer urgency (Claude's discretion per D — confirmed)

Icon/color-driven, no text change (respects UX-01):
- Normal (>60s): timer digits in primary text `#0F172A`.
- Warning (≤60s): digits shift to amber `#D97706`.
- Critical (≤10s): digits red `#DC2626` + gentle pulse animation (scale/opacity, `prefers-reduced-motion` disables the pulse and keeps the color).

## Layout Contract

Phase-1-specific structural rules (consumed by planner/executor):

- **Admin panel:** single scrollable column. Top: session/timer control bar (Display timer + accent primary CTA + secondary controls). Below: registration block (pre-session) OR monitoring grid (in-session) — the 4-6 team cards in a responsive grid, `lg` (24px) gap.
- **Team card:** reserves fixed space for (a) team name, (b) connection status dot+icon, (c) a **progress region placeholder** — empty in Phase 1 but laid out now so Phase 2 does not re-flow the card (D-08).
- **Team screen states (mutually exclusive full-screen states):**
  1. **Waiting** (pre-first-phase, D-05): full-screen centered, team name (Heading) + "Connectat ✓, esperant el professor" (Body, muted). No split layout visible.
  2. **Interstitial** (between phases, D-06): full-screen, ~1–2s, phase-colored "Ara: Fase {X}" with the phase color as accent.
  3. **Active split:** left action panel + right live preview (`iframe` sandbox, empty srcdoc shell in Phase 1). Grid split recommended ~40% panel / 60% preview; panel min-width protects controls.
- **Frozen state (D-11):** at timer zero the active split panel visually locks (reduced-opacity overlay + lock icon, no text) until admin advances; work is preserved as-is, no auto phase change.

## Copywriting Contract

Language: **Catalan** (all UI copy).

| Element | Copy |
|---------|------|
| Primary CTA | "Iniciar Fase" (pre-session) / "Següent fase" (between phases) |
| Empty state heading | "Cap equip registrat" |
| Empty state body | "Afegeix 4-6 noms per començar la sessió." |
| Error state | "Equip desconnectat" + "Esperant reconnexió automàtica…" |
| Destructive confirmation | Resync: "Resincronitzar equip? Recarregarà la seva pantalla immediatament." — confirm button "Sí, resincronitza", cancel "Cancel·la" |

Additional fixed copy for this phase:
- Team waiting screen: "Connectat ✓, esperant el professor" (D-05).
- Inter-phase interstitial: "Ara: Fase {HTML|CSS|JS}" (D-06).
- Team-selection prompt (per PC on first load): "Quin equip sou?" with the list of remaining unclaimed team names (claimed teams disappear, D-03).

**Destructive-action inventory (Phase 1):** Resync is the ONLY destructive-styled action (forces a full page reload on the target team's PC, D-09). No "release/delete team" action exists in v1 (D-04). Registration, Iniciar Fase, Pausar/Reprendre, and +1 minut are non-destructive and require no confirmation.
