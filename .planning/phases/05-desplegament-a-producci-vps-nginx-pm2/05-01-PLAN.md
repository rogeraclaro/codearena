<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| git repo → disc/VPS | El secret `ADMIN_SECRET` viu en un `.env` que mai ha d'entrar al control de versions |
| entorn de procés → codi app | PM2/shell estableixen variables d'entorn que el codi llegeix; una precedència errònia de PORT trenca el desplegament |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-01 | Information Disclosure | `.env` amb `ADMIN_SECRET` comès a git | high | mitigate | Afegir `.env` a `.gitignore` (Task 2) ABANS de crear cap `.env` al VPS — tanca T-04.1-05 costat repo |
| T-05-04 | Denial of Service (availability) | Precedència de PORT (ecosystem `env` vs `.env`) | medium | mitigate | Eliminar PORT de la config d'entorn de ecosystem.config.cjs; `.env` font única (Task 2, Pitfall 1) |
| T-05-SC | Tampering | `npm install dotenv` (cadena de subministrament) | high | mitigate | dotenv 17.4.2 amb veredicte de legitimitat OK (RESEARCH §Package Legitimacy Audit: ~137M/setmana, sense postinstall, no deprecated) — install directe sense checkpoint |
</threat_model>
