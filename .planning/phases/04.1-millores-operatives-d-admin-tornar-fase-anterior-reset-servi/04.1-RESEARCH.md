# Phase 04.1: Millores operatives d'Admin (tornar fase anterior, reset servidor) + preview CSS live code overlay - Research

**Researched:** 2026-07-06
**Domain:** Socket.io admin control flow · PM2 process supervision · live DOM/CSS overlay (vanilla JS)
**Confidence:** HIGH for the two in-codebase features (Fase anterior, CSS overlay); MEDIUM for the PM2/Reset feature (new operational dependency, one ESM-loader risk to verify at execution time)

## Summary

Aquesta fase són **tres features independents** empeltades sobre codi ja madur i molt consistent. Dues (Fase anterior, CSS overlay) són extensions gairebé mecàniques de patrons ja existents al repo — el codi ja té totes les peces necessàries i la feina és replicar-les simètricament. La tercera (Reset via PM2) introdueix una **dependència operativa nova** (PM2) que ara mateix **NO està instal·lada** a la màquina de dev, i porta un únic risc tècnic real que cal verificar aviat: PM2 carregant un entrypoint ESM.

**Fase anterior** és el cas més net: `gameState.nextPhase()` ja té tota la lògica; només cal el seu mirall `previousPhase()`, que ha de reutilitzar `startPhase()` (que ja dóna timer nou —D-02— i mai toca `doneAt` —D-03—). L'event, l'handler i el botó són còpies simètriques d'`ADMIN_NEXT_PHASE` i del CTA "Següent fase".

**Reset** ha de fer un restart REAL del procés (D-04): com que tot l'estat viu en memòria (`gameState`, `sessionStore`), `process.exit(0)` + PM2 revive = **wipe total** de la partida (equips, tokens, fase) — que és exactament el "reset" desitjat. El fitxer d'ecosistema PM2 **ha de ser `.cjs`** perquè `package.json` té `"type": "module"` (confirmat). Cal verificar aviat que PM2 pot arrencar l'entrypoint ESM `src/server/index.js`; si falla amb `ERR_REQUIRE_ESM`, la mitigació documentada és un llançador CJS d'una línia.

**CSS overlay** es construeix 100% de dades ja existents: `CSS_HOLES` agrupats per `hole.group` donen el bloc sencer del grup (D-08); l'etiqueta HTML (D-09) es deriva amb els helpers `pieceLabel()`/`containerLabel()` ja presents a `robotTemplate.js`; el patró d'overlay ancorat i el fade ja existeixen (`.frozen-overlay`, `transition: … 0.2s`).

**Primary recommendation:** Planifica-ho com **3 plans paral·lelitzables** (no depenen entre si). Prioritza un **checkpoint primerenc de verificació PM2** (instal·lar PM2 + `pm2 start ecosystem.config.cjs` real) abans de construir el botó Reset, perquè és l'únic punt amb incertesa d'entorn. Les altres dues features són baix risc i totes les peces ja hi són.

### Pattern 1: Mirall simètric de `nextPhase` → `previousPhase` (D-02, D-03)
**What:** Nova funció a `gameState.js` idèntica a `nextPhase` però decrementant l'índex.
**When to use:** Feature 1.
**Example:**
```javascript
// Source: src/server/gameState.js (mirall exacte de nextPhase, línia 309)
// Retorna false si ja és la primera fase (html) → no fa res, com nextPhase a l'última.
// Reusa startPhase → timer nou (D-02) i MAI toca team.doneAt (D-03) ni
// placement/cssValues/jsRules. Els equips amb doneAt.css/js segueixen congelats.
function previousPhase(durationMs) {
  const currentIndex = state.phase ? PHASE_ORDER.indexOf(state.phase) : -1;
  const prevIndex = currentIndex - 1;
  if (prevIndex < 0) return false; // ja a la primera fase (o cap fase) — no-op
  return startPhase(PHASE_ORDER[prevIndex], durationMs);
}
// … exportar a l'objecte gameState { …, previousPhase }
```
Handler simètric (mirall d'`ADMIN_NEXT_PHASE`, **sense** el bloc de partial-ranking — retrocedir no tanca cap fase):
```javascript
// Source: src/server/socketHandlers.js (mirall de línia 177, simplificat)
socket.on(EVENTS.ADMIN_PREV_PHASE, safeHandler((payload) => {
  if (!socket.rooms.has('admin')) return;         // V4: re-valida room server-side
  const durationMs = payload?.durationMs;
  if (!(Number.isFinite(durationMs) && durationMs > 0)) return;
  if (gameState.previousPhase(durationMs)) {
    io.to('session').emit(EVENTS.SESSION_FULL_STATE, gameState.getPublicState());
  }
}));
```

### Pattern 2: Reset via `process.exit` + PM2 (D-04, D-06)
**What:** Handler admin-only que avisa clients (opcional) i mata el procés; PM2 el revive net.
**When to use:** Feature 2.
**Example:**
```javascript
// Source: nou handler a src/server/socketHandlers.js — mateixa disciplina que
// ADMIN_FINALIZE_GAME (admin-only re-validat, safeHandler). 'io' ja és a l'scope
// de registerSocketHandlers, així que es pot tancar netament abans de sortir.
socket.on(EVENTS.ADMIN_RESET_SERVER, safeHandler(() => {
  if (!socket.rooms.has('admin')) return;                 // V4/T-04-01
  io.to('session').emit(EVENTS.SERVER_RESTARTING);        // avís amable (Claude's Discretion)
  // Deixa que el broadcast surti; després tanca sockets i surt. PM2 el revive.
  setTimeout(() => {
    io.close();                                           // disconnect net → clients auto-reconnect
    process.exit(0);
  }, 150);
}));
```

### Pattern 3: CSS overlay — bloc sencer del grup + etiqueta HTML (D-08, D-09)
**What:** Construir el text del bloc CSS del grup actiu des de `CSS_HOLES`, més l'etiqueta HTML de l'element.
**When to use:** Feature 3.
**Example:**
```javascript
// Source: derivat de src/client/client.js renderCssPanel (byGroup, línia 1074) +
// src/shared/robotTemplate.js pieceLabel/containerLabel.
// Mapa group(selector) → etiqueta HTML literal (D-09). Els helpers ja existents
// donen el tag real; aquest mapa només tria quin helper i amb quina clau.
import { pieceLabel, containerLabel } from '../shared/robotTemplate.js';
const GROUP_ELEMENT_LABEL = {
  '.antena':          () => pieceLabel('antena-esquerra'),   // <div class="antena">
  '.orella':          () => pieceLabel('orella-esquerra'),   // <img …class="orella">
  '.contenidor-ulls': () => containerLabel('contenidor-ulls'),// <div class="contenidor-ulls">
  '.ull':             () => pieceLabel('ull'),               // <span class="ull">
  '#robot-cap':       () => containerLabel('robot-cap'),     // <div id="robot-cap">
  '#nas':             () => pieceLabel('nas'),               // <button id="nas">
  '#boca':            () => pieceLabel('boca'),              // <output id="boca">
};

// Bloc CSS sencer del grup (D-08), amb el valor VIU per forat.
// liveValues = { ...latestCssValues, [activeHoleId]: valorQueS'estàCanviant }
function buildOverlayBlock(group, liveValues) {
  const rows = Object.entries(CSS_HOLES).filter(([, h]) => h.group === group);
  const lines = rows.map(([id, h]) => {
    const val = liveValues[id] ?? h.default;   // mateixa font que renderForatRow
    return `  ${h.prop}: ${val};`;
  });
  return `${group} {\n${lines.join('\n')}\n}`;  // pintar com a text monospace, NEVER innerHTML de dades
}
```

Recomanat: etiqueta neta `<img class="orella">` (menys soroll). `[ASSUMED]` — confirmar amb l'usuari el format exacte de l'etiqueta orella.

### Anti-Patterns to Avoid
- **Emetre `session:full-state` per al reset:** el reset no és un canvi d'estat de joc difusible — és un cicle de vida de procés. Prou amb l'avís opcional `SERVER_RESTARTING` i deixar que l'auto-reconnect faci la resta.
- **Recuperar `remainingMsAtPause` en retrocedir:** viola D-02. `startPhase` ja el posa a null — NO afegeixis lògica de recuperació de timer.
- **Reiniciar/desbloquejar `doneAt` en retrocedir:** viola D-03. No toquis `team.doneAt` enlloc del camí prev-phase.
- **`innerHTML` amb dades a l'overlay:** el bloc CSS és text; usa `textContent` (o construeix nodes). Mantén la disciplina GAME-06 de la resta del panell CSS (mai interpretar dades com a markup).
- **`ecosystem.config.js` (extensió `.js`):** fallarà amb `ERR_REQUIRE_ESM` per `type: module`. Ha de ser `.cjs`.
- **Overlay que capta punters:** ha de tenir `pointer-events: none` (com `.frozen-overlay`) per no interferir amb el robot ni els controls.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Restart del procés | Un script custom de watch/respawn (fork + re-spawn a mà) | PM2 `autorestart` | PM2 ja és la decisió del projecte; gestiona backoff, crash-loop guard i start-on-boot. |
| Reconnexió del client post-reset | Un protocol de "servidor tornarà, reintenta" a mà | Auto-reconnect natiu de socket.io-client (ja actiu, `reconnection` per defecte `true`) | El client ja es reconnecta sol; el codi de `connection` ja rehidrata l'estat. Verificat: `io({ transports:['websocket'] })` sense `reconnection:false`. |
| Confirmació d'accions d'admin | Un modal nou des de zero | El patró `<dialog class="confirm-dialog">` de `showFinalizeConfirm`/`showResyncConfirm` | Ja estilitzat, ja gestiona Escape i cleanup del node. |
| Etiqueta HTML de l'element (D-09) | Un mapa hardcoded selector→`<tag>` escrit a mà | `pieceLabel()`/`containerLabel()` de `robotTemplate.js` | Deriven el tag literal de la font única (SLOTS/CONTAINERS), sense duplicar coneixement (D-01). |
| Bloc CSS del grup (D-08) | Una llista de props escrita a mà per grup | Filtrar `CSS_HOLES` per `hole.group` (mateix `byGroup` de `renderCssPanel`) | `CSS_HOLES` és la font única; qualsevol canvi de forats es reflecteix sol. |

**Key insight:** Cap de les tres features necessita infraestructura nova de codi. La feina és **replicar patrons existents amb precisió simètrica**, no inventar-ne de nous. L'únic element genuïnament nou és el fitxer de config PM2.

## Common Pitfalls

### Pitfall 1: PM2 no pot carregar l'entrypoint ESM (`ERR_REQUIRE_ESM`)
**What goes wrong:** `src/server/index.js` és ESM (`import express …`) perquè `package.json` té `"type": "module"`. En algunes versions/configs, PM2 en fork mode intenta `require()` l'script i peta amb `ERR_REQUIRE_ESM`.
**Why it happens:** El loader intern de PM2 usa `require()`; un fitxer `.js` sota `type: module` és ESM i no és `require`-able.
**How to avoid:** Verifica-ho **abans** de construir el botó (checkpoint d'entorn): `npm i -g pm2@7.0.3 && pm2 start ecosystem.config.cjs && pm2 logs`. Si arrenca net → cap acció. Si peta amb `ERR_REQUIRE_ESM`, mitigació documentada: llançador CJS d'una línia `server.cjs` → `import('./src/server/index.js')` i apunta-hi `script: 'server.cjs'`. `[CITED: github.com/Unitech/pm2/issues/4540, 5451]`
**Warning signs:** `pm2 logs` mostra `ERR_REQUIRE_ESM` i el procés en estat `errored`/looping.

### Pitfall 2: Config PM2 amb extensió `.js`
**What goes wrong:** `pm2 start ecosystem.config.js` falla immediatament amb `ERR_REQUIRE_ESM` (el config, no l'app).
**Why it happens:** Mateix motiu que Pitfall 1, aplicat al fitxer de config.
**How to avoid:** Anomena'l `ecosystem.config.cjs`. `[VERIFIED: web + pm2 docs]`
**Warning signs:** Error en el mateix `pm2 start`, abans que l'app arrenqui.

### Pitfall 3: Crash-loop guard de PM2 si es prem Reset repetidament
**What goes wrong:** Si `min_uptime` és massa alt o el procés mor de seguida, PM2 pot marcar l'app com a `errored` després de `max_restarts`.
**Why it happens:** PM2 compta com a crash tot procés que mor abans de `min_uptime`.
**How to avoid:** `min_uptime: '5s'` + `max_restarts: 30`. En ús real (classe) el Reset es prem esporàdicament; el procés viu molt més de 5s entre resets, així que mai s'apropa al límit. Documenta que espamejar Reset molt ràpid pot pausar l'app (recuperable amb `pm2 restart codearena`).
**Warning signs:** `pm2 status` mostra `errored` i `pm2 logs` "script … exited too quickly".

### Pitfall 4: El Reset esborra tot l'estat — comportament CORRECTE però cal comunicar-ho
**What goes wrong:** Després del Reset, els equips perden el token i tornen a la pantalla de selecció; la partida arrenca de zero (cap equip registrat).
**Why it happens:** Tot l'estat (`gameState`, `sessionStore`, sessions de `connectionStateRecovery`) viu en memòria del procés; morir = wipe total. Això **és** el "restart REAL" de D-04.
**How to avoid:** No és un bug — és el requisit. Assegura't que la UAT/verificació ho documenta com a esperat (Reset = tornar a registrar equips). L'avís `SERVER_RESTARTING` fa que la transició es vegi amable en lloc d'un disconnect cru.
**Warning signs:** Algú ho reporta com a "bug: s'han perdut els equips" — és el disseny.

### Pitfall 5: Botó "Fase anterior" visible a la fase HTML (primera fase)
**What goes wrong:** Prémer "Fase anterior" a `html` no té destí (índex 0). `previousPhase` retorna `false` (no-op), però el botó semblaria trencat.
**Why it happens:** `html` és la primera de `PHASE_ORDER`.
**How to avoid:** Mostra el botó només quan `state.phase && state.phase !== 'html' && !finished`. Simètric a com el CTA d'avanç canvia segons la fase.
**Warning signs:** Clic sense efecte a la fase HTML.

## Security Domain

> `security_enforcement: true`, ASVS level 1. Aquesta fase afegeix accions d'admin d'alt impacte (retrocedir de fase, matar el procés) → el control d'accés és el punt calent.

### Applicable ASVS Categories

| ASVS Category | Applies | Standard Control |
|---------------|---------|-----------------|
| V2 Authentication | yes | Els nous events admin depenen de l'auth admin existent (`ADMIN_SECRET` al handshake). Cap canvi, però el Reset **ha de mantenir** `ADMIN_SECRET` a l'entorn de PM2 post-restart (si no, admin queda obert — el codi ja avisa amb `console.warn`). |
| V4 Access Control | yes | **Crític.** Cada nou handler (`ADMIN_PREV_PHASE`, `ADMIN_RESET_SERVER`) HA de re-validar `socket.rooms.has('admin')` server-side abans d'actuar — mai confiar en un flag del client (patró ja establert a tots els handlers admin). El Reset és especialment sensible: un client no-admin no pot poder matar el procés (DoS). |
| V5 Input Validation | yes | `ADMIN_PREV_PHASE` valida `durationMs` finit i >0 (mirall de next-phase). `ADMIN_RESET_SERVER` no porta payload → res a validar. L'overlay CSS usa `textContent` (mai `innerHTML` de dades). |
| V6 Cryptography | no | No s'introdueix cap operació criptogràfica nova. |

### Known Threat Patterns for aquest stack

| Pattern | STRIDE | Standard Mitigation |
|---------|--------|---------------------|
| Client no-admin dispara `ADMIN_RESET_SERVER` (mata el procés) | Denial of Service / Elevation | Re-validar room `admin` server-side dins el handler (fail-closed), igual que tots els handlers admin existents. |
| Espameig de `ADMIN_RESET_SERVER` per fer crash-loop | Denial of Service | Guard de PM2 (`min_uptime`/`max_restarts`) + l'acció ja és admin-only. Opcional: debounce al client. |
| `ADMIN_SECRET` absent després del restart PM2 | Elevation | Injectar `ADMIN_SECRET` via `env` de l'ecosystem/secret manager en prod; el `console.warn` existent el fa visible si falta. |
| Injecció via text de l'overlay CSS | Tampering / XSS | `textContent`/nodes, mai `innerHTML` amb valors; els valors ja passen la validació V5 de `CSS_HOLES` abans d'arribar aquí. |
