# Phase 04.1 — Security

> Per-phase security contract: threat register, accepted risks, and audit trail.

---

## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| client (admin browser) → Socket.io server | Events `ADMIN_PREV_PHASE` i `ADMIN_RESET_SERVER` arriben des d'un socket; el rol admin s'ha de re-validar server-side, mai confiar en el client. | payload `{durationMs}` (prev-phase) / cap payload (reset) |
| npm registry → màquina (dev/prod) | Instal·lació global del paquet `pm2` (dependència d'infra, no de projecte). | binari executable amb privilegis del sistema operatiu de l'usuari |
| dades de forat (valors CSS) → DOM de la preview | Els valors CSS es renderitzen a l'overlay en directe; s'ha d'evitar interpretar-los com a markup. | valors numèrics/text de `CSS_HOLES` |
| procés Node → PM2 (supervisor) | El procés senyala «surt ara» via `process.exit(0)`; PM2 el revive. Responsabilitat de cicle de vida delegada a l'infra. | cap dada — senyal de cicle de vida del procés |

## Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation | Status |
|-----------|----------|-----------|----------|-------------|------------|--------|
| T-04.1-01 | Elevation/Tampering | handler `ADMIN_PREV_PHASE` (socketHandlers.js) | high | mitigate | `if (!socket.rooms.has('admin')) return;` com a primera línia del handler (fail-closed) — verificat present a `socketHandlers.js` i cobert per `test/prevPhase.test.js` (`PREV-ADMIN-ONLY`) | closed |
| T-04.1-02 | Tampering | payload `durationMs` | medium | mitigate | Validació `Number.isFinite(durationMs) && durationMs > 0` abans d'actuar — verificada present i coberta per `test/prevPhase.test.js` (`PREV-INVALID-DURATION`) | closed |
| T-04.1-SC | Tampering | `npm install -g pm2` (supply-chain) | high | mitigate | Checkpoint blocking-human executat: llegitimitat de `pm2@7.0.3` verificada a npmjs.com (repo `github.com/Unitech/pm2`, sense `postinstall` sospitós, milions de descàrregues) i instal·lació fixada (no `latest`), aprovada explícitament per l'usuari | closed |
| T-04.1-07 | Denial of Service | crash-loop de PM2 | low | accept | `min_uptime: '5s'` + `max_restarts: 30` a `ecosystem.config.cjs` com a guard — verificat present; recuperable amb `pm2 restart codearena` | closed |
| T-04.1-03 | Denial of Service/Elevation | handler `ADMIN_RESET_SERVER` (socketHandlers.js) | critical | mitigate | `if (!socket.rooms.has('admin')) return;` com a primera línia (fail-closed) — verificat present i **provat en viu** pel verificador de fase (un socket no-admin no va disparar el reset) | closed |
| T-04.1-04 | Denial of Service | espameig de Reset → crash-loop | medium | mitigate | Guard de PM2 (`min_uptime`/`max_restarts`, compartit amb T-04.1-07) + acció admin-only (T-04.1-03) | closed |

No aplica en dev. Documentat al pla original. | closed |
| T-04.1-06 | Tampering (XSS) | render del bloc CSS de l'overlay (client.js) | medium | mitigate | Assignació via `textContent` (46 usos confirmats a `client.js`, `white-space: pre` a CSS); cap `innerHTML` real trobat (només comentaris que en documenten l'absència explícita) | closed |
