# Phase 04.1: Millores operatives d'Admin (tornar fase anterior, reset servidor) + preview CSS live code overlay - Context

**Gathered:** 2026-07-06
**Status:** Ready for planning

<domain>
## Phase Boundary

Aquesta fase afegeix tres eines operatives/pedagògiques sorgides durant les UAT manuals de la Fase 03, sense tocar l'abast del joc en si:

1. Botó "Fase anterior" al panell Admin, simètric a "Següent fase".
2. Botó "Reset" al panell Admin que reinicia el procés servidor (restart real, no només l'estat en memòria).
3. Overlay de codi CSS en directe a la preview de l'equip durant la Fase CSS, mostrant el forat actiu que s'està tocant.

</domain>

<decisions>
## Implementation Decisions

### Botó "Fase anterior" (Admin)
- **D-01:** Requereix confirmació ("Segur que vols tornar a la Fase X?") abans d'executar-se — evita repetir en sentit invers l'error que va motivar aquest tod́o.
- **D-02:** En retrocedir, el timer de la fase reoberta és nou (temps complet de la fase), mai un intent de recuperar `remainingMsAtPause` d'abans.
- **D-03:** `team.doneAt[phase]` de la fase reoberta NO es reinicia ni es desbloqueja — es manté igual. Un equip que ja havia premut "Finalitzar" continua congelat encara que l'admin torni enrere. `doneAt` és només un timestamp de referència pel scoring (D-05/D-06 de scoring.js), mai un gate reversible.

### Reset de servidors (Admin)
- **D-04:** El botó "Reset" fa un restart REAL del procés (no només neteja de l'estat de joc en memòria), amb confirmació prèvia (mateix patró que D-01: "Segur que vols reiniciar el servidor?").
- **D-05:** Disponible tant en dev com en producció (no restringit per entorn) — precisament perquè cal com a eina d'emergència durant una classe real.
- **D-06:** El mecanisme de restart és el mateix en dev i en producció: PM2 supervisa el procés en tots dos entorns (el botó fa que el procés surti amb un exit code, i PM2 el torna a aixecar). No hi ha dues rutes de codi diferents segons `NODE_ENV`.

### Preview CSS live code overlay
- **D-07:** Mostra NOMÉS el forat CSS que s'està tocant en aquell moment (no un històric de forats tocats recentment). Apareix/desapareix lligat a la interacció (input/focus) del control.
- **D-08:** Format: el bloc CSS sencer del grup del forat actiu (no una única línia `selector { prop: valor; }`) — totes les propietats del grup (p.ex. tot `#robot-cap { ... }`), amb el valor que s'està canviant actualitzant-se en directe.
- **D-09:** A sobre del bloc de selector+codi, es mostra també l'etiqueta HTML concreta que s'està modificant (p.ex. `<div id="robot-cap">`), per reforçar la connexió element↔selector↔codi.
- **D-10:** Transició d'entrada/sortida: fade suau curt (~150-200ms), no instantani — consistent amb la resta de transicions ja usades al joc.

### Claude's Discretion
- Format visual exacte del requadre (mida, tipografia monospace, posicionament exacte ancorat a la part inferior de la preview) — el disseny general ja estava fixat pel todo original; els detalls de mida/espaiat queden a criteri de la implementació, sempre respectant "mínim soroll visual" i sense tapar el robot.
- Mecanisme exacte de com el botó "Reset" fa sortir el procés (codi de sortida, event abans de tancar per netejar sockets) — decisió tècnica que no canvia el comportament observable acordat (D-04/D-06).

## Existing Code Insights

### Reusable Assets
- Patró de CTA/botó d'Admin ja establert a `admin.js` (~línia 401) per a "Següent fase" — el botó "Fase anterior" n'és simètric.
- `CSS_HOLES`/`CSS_GROUP_ORDER` a `robotTemplate.js` ja agrupen els forats per selector — base directa per construir el bloc CSS sencer del grup a l'overlay.

### Established Patterns
- `doneAt` com a timestamp pur de referència (mai gate reversible) — patró ja establert al motor de scoring (`src/shared/scoring.js`, D-05/D-06 de scoring); D-03 d'aquesta fase l'estén explícitament al cas de retrocedir de fase.
- Confirmacions abans d'accions destructives/irreversibles ja és un patró implícit del projecte (accions d'Admin que afecten tots els equips en directe).

### Integration Points
- Botó "Fase anterior" crida un nou event Admin simètric a `ADMIN_NEXT_PHASE`, consumit per una nova funció simètrica a `nextPhase` a `gameState.js`.
- Botó "Reset" necessita un nou event Admin + lògica de procés (exit controlat) al servidor, i configuració PM2 aplicada també en dev (ecosystem file).
- Overlay CSS s'integra al flux existent de `renderForatRow`/`applyCssHole` a `client.js`, afegint el bloc visual en resposta als mateixos events d'interacció (input/focus) que ja disparen el canvi de CSSOM.
