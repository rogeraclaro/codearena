## Accomplishments
- `gameState.previousPhase(durationMs)` added as an exact mirror of `nextPhase()` (decrements `PHASE_ORDER` index instead of incrementing), reusing `startPhase()` so D-02 (fresh timer) and D-03 (never touches `team.doneAt`) are satisfied without any extra logic.
- `EVENTS.ADMIN_PREV_PHASE` declared as the single source of truth for the new socket intent; the admin-only handler in `socketHandlers.js` re-validates `socket.rooms.has('admin')` (fail-closed, T-04.1-01) and validates `durationMs` (T-04.1-02) before broadcasting `SESSION_FULL_STATE`.
- Admin panel gets a "Fase anterior" button, visible only when a phase is active, it isn't HTML, and the game hasn't finished (Pitfall 5 guard); clicking it asks "Segur que vols tornar a la Fase X?" before emitting the event.
- `showResyncConfirm`/`showFinalizeConfirm`'s duplicated `<dialog class="confirm-dialog">` blocks were collapsed into a single reusable `showConfirm(message, confirmLabel, destructive)` helper — one dialog-construction site instead of three.
- `test/prevPhase.test.js` added covering the no-op at HTML, the fresh-timer behavior, `doneAt` preservation across a retreat, and the admin-only/invalid-duration socket guards.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed a stray-broadcast test flake in test/prevPhase.test.js's own setup**
- **Found during:** Task 1, while verifying the RED→GREEN transition
- **Issue:** The `TEAM_SELECT` handler broadcasts `SESSION_FULL_STATE` to the `session` room right after claiming a team; the test's setup didn't attach a listener for it. Because the tests immediately following are synchronous `gameState`-direct calls with no `await` between them, the event loop never got a chance to process that already-in-flight packet — it was later delivered (as a misleading stray event) only once the first real `await` (a socket round-trip test) forced an event-loop turn, causing a false failure in `PREV-ADMIN-ONLY`.
- **Fix:** Added a short `await wait(50)` at the end of the setup test to drain any in-flight broadcasts before the synchronous model-level tests run.
- **Files modified:** test/prevPhase.test.js
- **Verification:** `node --test test/prevPhase.test.js` — all 8 subtests pass in under 1 second (no more spurious 60s test-timeout hangs).
- **Committed in:** fa5d125 (Task 1 GREEN commit, alongside gameState.js/events.js)
