# Phase 4 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope of this contract:** the two NEW UI surfaces introduced by scoring/ranking, plus three changes to the existing per-phase "Finalitzar" affordance:

1. **Team final results screen** — a new full-screen state (`state.phase === 'results'` / a terminal `finished` flag) shown to **all** teams simultaneously when the Admin fires ADMIN-07. Shows the **global ranking + global percentage of every team**, but the **sub-check detail is visible only for the current team's own work** — never another team's (D-10, D-11).
2. **Admin partial mini-ranking** — an **Admin-only** compact ranking rendered in the Admin panel each time a game phase closes (HTML→CSS, CSS→JS). Teams never see it (D-12, D-13). Uses the same scoring pipeline with not-yet-played phases counting as 0.
3. **"Finalitzar" affordance changes** (GAME-08 retrofit): HTML button gains a **100%-correctness gate** (D-07), the CSS button is **removed** (D-08), JS has **no button** (D-09). The Admin CTA at the last phase becomes **"Finalitzar i Mostrar Resultats"** (ADMIN-07).

**Inheritance:** this contract EXTENDS the Phase 1–3 UI-SPECs. All existing tokens in `src/client/shared/tokens.css`, the LIGHT theme, the 60/30/10 surface split, phase color-coding (`--phase-html/css/js`), the typography roles, the spacing scale, the monospace code-label pattern, the motion tokens, the frozen overlay, the Admin `.team-grid`/`.team-card`, the `.btn`/`.btn-accent` system, and the `.confirm-dialog` pattern are reused verbatim. **This phase adds no new design tokens** — every new surface maps onto existing `var(--*)` tokens. Executor MUST reference existing tokens, never hard-coded literals (Phase 1–3 token strategy).

**Audience driver (rectora):** unchanged — students with zero prior experience, in a 15–20 min live classroom dynamic. The results screen is a **pedagogical debrief** (SCORE-05), not a leaderboard shaming ritual: a team's own concrete misses stay private to its own screen (D-10), the ranking shows only names + global % publicly (D-11), and a missed sub-check is rendered as **neutral/incomplete, never as a red error** (project no-frustration ethos, consistent with Phase 2/3 "state, never error color").

## Spacing Scale

Reuses the inherited `--space-*` scale unchanged (4 / 8 / 16 / 24 / 32 / 48 / 64). No new spacing tokens.

| Token | Value | Phase-4 usage |
|-------|-------|---------------|
| xs | 4px | Gap between a rank number and its medal/trophy icon; gap between a sub-check icon and its label |
| sm | 8px | Gap between sub-check items within a group; inner gap of a rank row's meta cluster |
| md | 16px | Gap between rank rows; gap between sub-check groups (HTML/CSS/JS); Admin mini-rank row spacing; `.team-card` inner padding (inherited) |
| lg | 24px | Rank row inner padding; sub-check group inner padding; `.team-grid` gap (inherited) |
| xl | 32px | Results-screen outer padding; vertical separation between the ranking block and the own-detail block |
| 2xl | 48px | Vertical breathing room above the results heading / below the winner row |
| 3xl | 64px | Top/bottom page padding of the centered results layout (mirrors `.empty-state` padding) |

Exceptions:
- **Hit target (`--hit-target-min: 44px`):** the Admin **"Finalitzar i Mostrar Resultats"** CTA and its confirm-dialog buttons render at ≥44px hit area (inherited `.btn` rule). The team results screen is **read-only** — it has no interactive controls, so no hit-target rule applies there.
- **Results layout width:** the centered ranking column is capped (recommended `max-width: 640px`, mirroring the `.admin-login` / `.confirm-dialog` capped-column convention) so rows don't stretch across a wide classroom projector. Not a spacing-scale value — a layout rule.

## Typography

Reuses the inherited roles unchanged — exactly 4 sizes, 2 weights. **No new sizes or weights.**

| Role | Size | Weight | Line Height | Token | Phase-4 usage |
|------|------|--------|-------------|-------|---------------|
| Body | 16px | 400 | 1.5 | `--font-size-body` | Sub-check item labels; the "Rànquing parcial" caption on the Admin mini-rank; secondary meta text |
| Label | 14px | 600 | 1.4 | `--font-size-label` | Sub-check group micro-headings (HTML / CSS / JS); Admin mini-rank team labels; rank-row rank number |
| Code label | 14px | 400 | 1.4 | `--font-family-mono` | Real identifiers inside a team's OWN sub-check detail — e.g. an HTML slot/tag name (`<antena>`), a CSS property (`border-color:`), a JS event/element token. Monospace, read-only, never editable (GAME-06 continuity) |
| Heading | 24px | 600 | 1.2 | `--font-size-heading` | Team name in each rank row; results-screen section heading ("Classificació", "El teu detall") |
| Display | 56px | 600 | 1.0 | `--font-size-display` | The current team's **own global percentage** shown large (its headline result); `#1` winner's percentage may also use Display. `font-variant-numeric: tabular-nums` (inherited on `.timer-display` pattern) for aligned digits |

Notes:
- **The percentage is the load-bearing number.** Global % renders in Display/Heading with tabular-nums so digits align down the ranking column. Rank position numbers use Label weight (600) so ordering reads at a glance without competing with the percentage.
- **Sub-check labels stay short.** Each item is an icon + a terse Catalan label (Body), optionally with a monospace real identifier — never a sentence. Icons carry the pass/miss meaning (UX-01).

## Color

Base theme: **LIGHT** (inherited). Reuses all Phase-1 color tokens. **No new color tokens.**

### Surface split (60/30/10) — inherited

| Role | Value | Phase-4 usage |
|------|-------|---------------|
| Dominant (60%) | `#F8FAFC` (`--color-bg`) | Results-screen canvas; rank-bar track; recessed inset behind sub-check groups |
| Secondary (30%) | `#FFFFFF` (`--color-surface`) | Rank rows, sub-check group cards, Admin mini-rank rows; borders `--color-border` |
| Accent (10%) | `#059669` (`--color-accent`) | **Admin only** — the **"Finalitzar i Mostrar Resultats"** primary CTA (inherits `.btn-accent`). NEVER used on the team results screen |
| Destructive | `#DC2626` (`--color-destructive`) | **Not used as an error signal this phase.** A missed sub-check is neutral, not destructive. `--color-destructive` remains only in its inherited role (Admin team-remove `.btn-destructive`, unrelated to scoring) |

**Accent reserved for:** the single Admin primary CTA ("Finalitzar i Mostrar Resultats") — same reservation as Phases 1–3, where accent-green never appears on any team screen. The team results screen has **zero accent-green elements**; the current team's own row is emphasized with a **neutral** treatment (see below), not accent.

### Highlighting the current team's own row (D-10/D-11)

The viewing team must instantly find itself in the ranking **without** spending the reserved accent budget:

| Signal | Value | Rule |
|--------|-------|------|
| Own-row left identity strip | `--color-text` (3px left border) | Marks "this is you" — neutral, high-contrast, mirrors the phase-identity strip pattern (`.css-forat-group` / `.js-rules`) but in neutral text color, not a phase hue |
| Own-row emphasis | `--font-weight-label` (600) on the name + `--color-bg` inset | Bolder weight + faintly recessed surface, no color change |
| Every other team's row | `--color-surface` on `--color-border`, name at Body weight | Plain, unemphasized |

### Rank position & podium (no new color tokens)

- **No gold/silver/bronze palette.** Introducing three medal colors would violate the "no new color tokens" contract and add visual noise (UX-01). Ordering is carried by the **rank number** (Label weight) and vertical position.
- **Winner marker (#1 only):** a Lucide **`Trophy`** icon in `--color-text`, inline before the #1 team name. A single monochrome trophy is a restrained "celebration" beat consistent with the icon-over-decoration ethos. Ranks #2+ show the rank number only, no icon.

### Percentage bar (`.rank-bar`) — at-a-glance comparison

Global % is authoritative as text; an optional horizontal bar aids scanning:

| Part | Value | Rule |
|------|-------|------|
| Track | `--color-border` | Full-width recessed rail (reuses the `.pip` track color) |
| Fill | `--color-text` | Neutral fill, width = global %. **Not** a phase hue — the results screen is cross-phase (global score), so no single phase color applies |

### Sub-check pass/miss icons (team's own detail, SCORE-05)

| State | Icon (Lucide) | Color | Rule |
|-------|---------------|-------|------|
| Passed sub-check | `Check` / `CircleCheckBig` | `--status-connected` (`#16A34A`) | Reuses the inherited "positive/done" family (same token as the connection-OK icon and the HTML finish button) |
| Missed sub-check | `Circle` (empty) or `Minus` | `--color-muted` (`#64748B`) | **Neutral, never red.** A miss is "not yet reached", not an error — no `--color-destructive`, no frustration (project ethos, Phase 2/3 continuity) |

No new semantic colors. The only place status-green appears on the team screen is the passed-sub-check icons in the team's OWN private detail block.

### Admin partial mini-ranking colors (D-12/D-13)

Rendered inside the Admin panel (Admin-only surface), reusing Admin `.team-card`/`.btn` token vocabulary: rows on `--color-surface`/`--color-border`, team labels `--color-text`, partial % `--color-muted` (signals "provisional"), optional thin bar with `--color-border` track + `--color-muted` fill. The "Rànquing parcial" caption names which phase just closed. Accent-green is **not** used here (reserved for the CTA).

## Motion Contract

Reuses the inherited motion tokens — `--motion-snap: 150ms`, `--motion-ease: cubic-bezier(0.2,0,0.2,1)`. **No new motion tokens.** All motion is decorative; every surface is fully usable with motion disabled.

| Interaction | Read | Spec |
|-------------|------|------|
| **Results screen appears** | "Els resultats han arribat" | The ranking column fades/rises in over `--motion-snap` `--motion-ease` (mirrors the `.js-rule` enter). Rows may stagger subtly; not required |
| **Percentage bar fill** | Score magnitude | Each `.rank-bar` fill animates its width from 0 → % over `--motion-snap` on first paint |
| **Own global % (Display)** | The team's headline number | Static by default. An optional count-up is allowed but must land on the exact value; skip under reduced motion |
| **Admin mini-rank on phase close** | "Here's where teams stand now" | Appears instantly on the phase-close broadcast; no entrance animation required (Admin ops surface, clarity over flourish) |

**Reduced motion:** wrap all of the above in `@media (prefers-reduced-motion: no-preference)` (same pattern as `.timer-display` critical pulse and the Phase-2/3 animations). With reduced motion, the ranking, bars, and percentage render at their final values **instantly** — no count-up, no fill animation. Functionality and legibility fully preserved.

## Component Inventory

All hand-rolled Vanilla JS + CSS tokens. Team surfaces live in `client.js`; the Admin mini-ranking lives in `admin.js`.

### Team final results screen (new render branch in `renderScreenForState`)

Triggered by a new terminal state (recommended: `state.phase === 'results'`, or a `state.finished === true` flag on the authoritative state). Full-screen, centered, read-only. Two stacked blocks:

| Component | Description | Key states |
|-----------|-------------|------------|
| **Results screen** (`.results-screen`) | Full-viewport LIGHT canvas, centered capped column, `--space-3xl` top/bottom padding. Contains the ranking block then the own-detail block | — |
| **Section heading** (`.results-heading`) | Heading role. e.g. "Classificació" for the ranking block, "El teu detall" for the private block | — |
| **Ranking list** (`.rank-list`) | Vertical, descending by global %. One `.rank-row` per team (all 4–6 teams, D-10) | — |
| **Rank row** (`.rank-row`) | `[rank #] [Trophy if #1] [team name — Heading] [rank-bar] [global % — tabular]`. The current team's row carries the neutral own-row emphasis (identity strip + bold + inset) | own-team · other-team · winner(#1) |
| **Rank bar** (`.rank-bar`) | Horizontal track (`--color-border`) + neutral fill (`--color-text`), width = that team's global % | — |
| **Own global %** (`.own-percent`) | The current team's global percentage in Display size — its headline outcome. Shown once, prominently, above or leading the own-detail block. **Global only, no per-phase split** (D-11) | — |
| **Own sub-check detail** (`.subcheck-detail`) | The pedagogical debrief (SCORE-05), **only for the viewing team** (D-10). Grouped by phase with color-coded phase badges | — |
| **Sub-check group** (`.subcheck-group`) | One card per phase (HTML / CSS / JS), headed by the inherited `.phase-badge` (`--phase-html/css/js`). Contains that phase's pass/miss items | — |
| **Sub-check item** (`.subcheck-item`) | `[Check/Circle icon] [terse Catalan label] [optional monospace identifier]`. Passed = `--status-connected` check; missed = `--color-muted` empty circle. No red, no per-item numeric score exposed | passed · missed |

**Privacy invariant (D-10):** the ranking block lists every team's name + global % (public info); the `.subcheck-detail` block renders **exclusively** the viewing team's own sub-checks. The client MUST NOT receive or render another team's sub-check breakdown. This is a data-shape contract as much as a visual one — the server payload for a team should include full ranking (names + global %) but sub-check detail only for `socket.data.teamId`.

### Admin partial mini-ranking (new element in `admin.js`)

## Layout Contract

### Team results screen

Full viewport, LIGHT theme, single centered capped column (`max-width: 640px`), `--space-3xl` vertical page padding — no split, no preview iframe (the game is over).

Top → bottom:
1. **Results heading** (Heading) — "Classificació".
2. **Ranking list** — all 4–6 `.rank-row`s, descending by global %, winner (#1) marked with the `Trophy` icon, the viewing team's row neutrally emphasized. Each row shows name + `.rank-bar` + global %.
3. **`--space-2xl` separation.**
4. **Own outcome** — the viewing team's own global % in Display ("El teu resultat"), then the **own sub-check detail** grouped by phase (HTML / CSS / JS) with phase-badge headers and pass/miss item lists. **Global % only, no per-phase % shown to the team** (D-11).

The screen is **read-only**: no buttons, no next action on the team side — the Admin owns the session lifecycle.

### Admin partial mini-ranking

Rendered within the existing Admin panel layout (`admin.js`), placed near the control bar / above or beside the `.team-grid` so the teacher reads standings alongside live team cards. Appears on each phase-close broadcast and updates to the final ranking on finalize. Compact list-with-bar (see Component Inventory); does not replace the `.team-grid`, complements it.

### Rendering / state wiring (integration notes for planner, non-binding on visuals)

- Team results are a **new branch** in `renderScreenForState` keyed on the terminal state; the Admin finalize event (ADMIN-07) is what pushes every team into it (Success Criterion 1: all screens flip instantly).
- The partial mini-ranking hooks the **same phase-transition point** as `admin:next-phase` / `nextPhase` in `gameState.js` (D-13), computing the shared scoring pipeline with unplayed phases = 0.
- Both surfaces consume one scoring output shape; the team payload is **filtered** so sub-check detail is per-`teamId` only (D-10 privacy invariant above).

## Copywriting Contract

Language: **Catalan** (all UI copy). Deliberately minimal (UX-01); the percentage number, ranking order, and pass/miss icons carry most of the meaning.

| Element | Copy |
|---------|------|
| Primary CTA — Admin (last phase) | **"Finalitzar i Mostrar Resultats"** (ADMIN-07) — replaces "Següent fase" when `state.phase === 'js'`; emits the finalize event |
| Finalize confirmation | Reuse `.confirm-dialog`. Message: **"Vols finalitzar la partida? Es mostraran els resultats a totes les pantalles."** · confirm: **"Finalitzar"** · cancel: **"Cancel·lar"** |
| Primary CTA — HTML team | **"Finalitzar"** (enabled only at 100%, D-07) → **"✓ Fet"** when done/disabled |
| Primary CTA — CSS / JS team | **None** — buttons removed (D-08/D-09), continuous scoring |
| Results heading — ranking | **"Classificació"** |
| Results heading — own block | **"El teu resultat"** (headline global %) + **"El teu detall"** (sub-check breakdown) |
| Rank row | team name + global % (e.g. **"72%"**). Winner marked by `Trophy` icon, no "guanyador" label needed |
| Sub-check group headers | inherited phase badges: **"HTML"** / **"CSS"** / **"JS"** |
| Sub-check item label | terse Catalan noun for the checked thing (e.g. **"Antena"**, **"Color del cap"**, **"Regla: click → nas"**) + optional monospace identifier; icon carries pass/miss |
| Admin mini-rank caption | **"Rànquing parcial — fase {HTML/CSS} tancada"** (names the just-closed phase; signals provisional) |
| Empty state | **None** — the ranking always has the registered 4–6 teams; the own-detail always has the three phase groups (a phase a team barely touched simply shows mostly-missed neutral items, not an empty state) |
| Error state | **None by design** — a missed sub-check is a neutral incomplete marker (`--color-muted`), never a red error. The gated HTML "Finalitzar" button stays silently disabled (the N/7 pips are the affordance), no explanatory error text |
| Destructive confirmation | The finalize confirm above is the only confirmation. It is **flow-irreversible, not data-destructive** — a light confirm, reusing the existing dialog, prevents an accidental early end to a 15–20 min session |

**Destructive-action inventory (Phase 4):** none that lose data. "Finalitzar i Mostrar Resultats" is terminal for the session flow (everyone moves to results) and therefore gated by a single confirmation dialog. Removing the CSS finish button (D-08) removes an affordance; it needs no user-facing copy.
