# Phase 3 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope of this contract:** the **playable CSS phase** and the **playable JS phase**, both rendered inside the same Phase-1/Phase-2 split shell (`renderActiveSplitScreen()` in `src/client/client.js`). It specifies the visual and interaction contract for:
- the **CSS "forat" panel** (`state.phase === 'css'`) — grouped real-CSS snippets with the value replaced by a closed control (color picker or slider), reproducing the definitive Bender design (D-01…D-10);
- the **JS rule-builder panel** (`state.phase === 'js'`) — addable rows reading as a natural-language sentence `Quan [event] a [origen] → a l'element [destí] → Fes [acció]`, with a per-row **"Veure"** preview button and a final **"Afegir JavaScript"** button (D-11…D-18);
- the **cross-phase 50/50 split correction** (D-21 — applies to html/css/js);
- the **Admin card progress** rule for these two phases: **connection status only, no numeric counter** (D-22).

**Inheritance:** this contract EXTENDS the Phase 1 and Phase 2 UI-SPECs. All existing tokens in `src/client/shared/tokens.css`, the 60/30/10 surface split, the phase color-coding (`--phase-css`, `--phase-js`), the typography roles, the spacing scale, the monospace code-label pattern, the motion tokens, and the frozen overlay are reused verbatim. Only NEW visual/interaction surfaces introduced by the CSS and JS games are specified here. Executor MUST reference existing `var(--*)` tokens, never hard-coded literals (Phase 1/2 token strategy).

**Audience driver (rectora):** unchanged — adults ~40 anys, zero prior experience, ~10 min prior talk. Every discretionary choice resolves toward minimal cognitive load, closed controls over free text (GAME-06), real syntax shown but never editable, and no-frustration. A control that targets an absent element is a silent no-op, never an error (D-19, GAME-07).

## Spacing Scale

Reuses the inherited `--space-*` scale unchanged (4 / 8 / 16 / 24 / 32 / 48 / 64). No new spacing tokens.

| Token | Value | Phase-3 usage |
|-------|-------|---------------|
| xs | 4px | Gap between a property label and its inline value readout; range-thumb inner spacing |
| sm | 8px | Gap between the 4 selects inside a JS rule row; gap between forat rows inside a selector group |
| md | 16px | Gap between selector groups (CSS) / between rule rows (JS); group inner padding |
| lg | 24px | Outer padding of each selector group frame (mirrors Phase-2 `.slot-frame`) |
| xl | 32px | Action-panel outer padding (inherited from `.action-panel`) |
| 2xl | 48px | Vertical breathing room above the "Afegir JavaScript" button |

Exceptions:
- **Hit target (reuses `--hit-target-min: 44px`):** every interactive control — the color swatch button, the range thumb, each `<select>`, the "Veure" button, the "Afegir JavaScript" button, and any row-remove affordance — renders at a **minimum 44px hit area**. Sliders get a ≥44px-tall interactive track region even if the visible rail is thinner.
- **Panel overflow:** the CSS panel (7 selector groups / 16 controls) and the JS panel (up to 6 rows) can exceed viewport height. `.action-panel` is constrained to the viewport and scrolls vertically (`max-height: 100vh; overflow-y: auto`); the right preview never scrolls. Not a spacing-scale value — a layout rule.

## Typography

Reuses the inherited roles unchanged — exactly 4 sizes, 2 weights. **No new sizes or weights.**

| Role | Size | Weight | Line Height | Token | Phase-3 usage |
|------|------|--------|-------------|-------|---------------|
| Body | 16px | 400 | 1.5 | `--font-size-body` | JS sentence connective words ("Quan", "a", "a l'element", "Fes") |
| Label | 14px | 600 | 1.4 | `--font-size-label` | (inherited) section micro-headings; Admin card status text |
| Code label | 14px | 400 | 1.4 | `--font-family-mono` | **All real CSS syntax** — selector open/close (`.antena {` / `}`), property names (`background-color:`), the trailing `;`, and the live value readout (`95px`, `#8a9ba8`). Monospace, read-only text, never an editable field (GAME-06). Same pattern as Phase-2 slot/frame labels |
| Heading | 24px | 600 | 1.2 | `--font-size-heading` | (inherited) team name in panel header |
| Display | 56px | 600 | 1.0 | `--font-size-display` | (inherited) synchronized timer |

Notes:
- **The forat IS real CSS, read as code.** A CSS control renders as a line of genuine CSS with only the value swapped for a control: `background-color: [■] ;` where `[■]` is the swatch, and `width: [——●——] 40px ;` where the slider sits inline and the monospace readout shows the live value. The property name, colons, semicolons, and selector braces are real syntax rendered as **plain monospace text nodes** — never interpreted, never editable (GAME-06, same anti-syntax guarantee as the Phase-2 tag labels).
- **The JS sentence is intentionally text.** D-12's "es llegeix com llenguatge natural" is the one place where readable words carry the meaning; the connective words are Body 16/400. Each `<select>` value is a short real token (`click`, `nas`, `#robot-cap`, `Canviar de color`). Keep selects textual; do not replace them with icon-only pickers.

## Color

Base theme: **LIGHT** (inherited). Reuses all Phase-1 color tokens. **No new color tokens.**

### Surface split (60/30/10) — inherited

| Role | Value | Phase-3 usage |
|------|-------|---------------|
| Dominant (60%) | `#F8FAFC` (`--color-bg`) | Action-panel canvas behind the groups/rows; inset background of a selector group frame |
| Secondary (30%) | `#FFFFFF` (`--color-surface`) | Forat rows, JS rule rows, `<select>` and swatch backgrounds; borders `--color-border` |
| Accent (10%) | `#059669` (`--color-accent`) | **Reserved — NOT used on the team screen.** Kept exclusively for the Admin primary CTA ("Següent fase") inherited from Phase 1 |
| Destructive | `#DC2626` (`--color-destructive`) | Not used this phase. Removing a JS rule row is encouraged undo, not a destructive action |

**Accent reserved for:** the single Admin primary action button only (inherited). The team CSS/JS screens have **no accent-green element**. NEVER apply accent-green to forats, sliders, selects, "Veure", or "Afegir JavaScript".

### Phase identity color (UX-02) — drives each sub-phase's feedback color

| Sub-phase | Identity token | Value | Where it appears |
|-----------|---------------|-------|------------------|
| CSS | `--phase-css` | `#264DE4` (blue) | Selector-group left identity strip; slider filled track + thumb; `<input type=color>` focus ring; phase badge + interstitial (inherited) |
| JS | `--phase-js` | `#D4B106` (mustard) | Rule-panel left identity strip; "Afegir JavaScript" button fill; phase badge + interstitial (inherited) |

- **CSS interactive affordance = `--phase-css` blue** (`#264DE4`) — high contrast on white, used for the slider fill/thumb and the color-input focus ring so the "live" control reads clearly.
- **JS interactive affordance stays neutral, identity color is decorative.** `--phase-js` (`#D4B106`) has **insufficient contrast against white for a load-bearing UI affordance** (< 3:1). Therefore JS `<select>` and "Veure" controls use **native/neutral styling** with a `--color-text` focus outline; `--phase-js` is used only as an **identity strip / badge / button-fill with a `--color-border` edge**, never as the sole indicator of an interactive state. See Checker note in Registry/Sign-off.
- **"Afegir JavaScript" button:** filled `--phase-js` with a 1px `--color-border` edge and `--color-text` label (dark text on mustard reads clearly, unlike white-on-mustard). This is the phase-consistent analogue of an accent CTA without spending the reserved accent-green budget.

### Neutral / empty states

| Element | Value | Usage |
|---------|-------|-------|
| Selector group frame border | `--color-border`, solid | Mirrors the Phase-2 `.slot-frame` scaffold look |
| Group inset background | `--color-bg` | Sits the real-CSS lines on a faintly recessed surface |
| Empty JS select (placeholder) | `--color-muted` text | An unfilled dropdown shows a dimmed `…` placeholder |
| Disabled JS option / disabled "destí" select (composite action, D-17) | `--color-muted` + reduced opacity | Anti-repetition (D-15) greys the already-used `(event, origen)` option; a composite action greys the whole `destí` select — both communicated by disabled state, **no error color/text** |

No new semantic colors. Invalid or blocked choices produce **no red/error signal** — disabled state and (where motion applies) a brief settle carry the message (UX-01, no-frustration). This matches Phase 2's "motion/state, never error text" rule.

## Motion Contract

Reuses the inherited motion tokens — `--motion-snap: 150ms`, `--motion-ease: cubic-bezier(0.2,0,0.2,1)`. **No new motion tokens.** All motion is decorative reinforcement; both games are fully usable with motion disabled.

| Interaction | Read | Spec |
|-------------|------|------|
| **Slider drag (CSS)** | Value changes live | Native range input; preview re-renders on `input` via the authoritative server round-trip (`--motion-snap` is not applied to the value itself — the live preview IS the feedback) |
| **Color pick (CSS)** | Swatch updates, preview updates | Native `<input type=color>`; swatch fill transitions `--motion-snap` `--motion-ease` |
| **Add rule row (JS)** | A new empty sentence appears | New `.js-rule` slides/fades in over `--motion-snap` `--motion-ease` |
| **"Veure" (JS)** | "Show me just this rule" | The button gives a brief pressed-state settle (`--motion-snap`); the actual effect plays inside the preview iframe |
| **Remove rule row (JS)** | Easy, unpunished undo | Row collapses out over `--motion-snap`; no confirmation dialog |

**Reduced motion:** wrap the swatch transition, the add-row slide-in, and the row-collapse in `@media (prefers-reduced-motion: no-preference)` (same pattern as the existing `.timer-display` critical pulse and Phase-2 snap animations). With reduced motion, rows appear/disappear and values update **instantly** — functionality preserved.

## Layout Contract

Inside the corrected `.active-split` (left `.action-panel` 50%, right `.preview-frame` 50%).

**50/50 split correction (D-21, cross-phase):** change `.active-split { grid-template-columns: minmax(280px, 40%) 1fr; }` to **`grid-template-columns: 1fr 1fr;`** (exact equal halves). Classroom hardware is fixed evergreen desktop, so the previous `minmax(280px, …)` floor is unnecessary; the panel instead gains vertical scroll for dense content. This correction applies to the **html phase too**, not only css/js — it is the single shared shell.

**Left action panel (50%), top → bottom:**
1. **Panel header** (inherited): phase badge (blue for CSS / mustard for JS) + team name (Heading) + synchronized timer (Display).
2. **Game body**, conditioned on `state.phase`:
   - **CSS:** the stack of **selector groups** (one per styled element), each a monospace `.selector {` … `}` frame containing its forat rows. Recommended top→bottom order follows the robot spatial reading: `.antena` → `.orella` → `.contenidor-ulls` → `.ull` → `#robot-cap` → `#nas` → `#boca` (planner may reorder for visual balance; discretion per D-Claude).
   - **JS:** the stack of **rule rows** followed by the **"Afegir JavaScript"** button (with ~48px breathing room above it).
3. Panel scrolls vertically (`max-height:100vh; overflow-y:auto`) when content exceeds the viewport; the preview half never scrolls.

**Right preview (50%):** the real rendered robot, inherited iframe. In the CSS phase the robot visibly acquires its Bender look as values are set; in the JS phase the robot is fully styled (carrying the CSS phase result) and reacts to "Veure"/hover/click. **No target thumbnail** — the teacher projects the Plana Model (consistent with Phase 2 D-14).

**DOM-incomplet (GAME-07 / D-19):** no special UI. A forat or JS rule whose target element is absent is a natural no-op. The **only** quality requirement is that such a control throws no console error and does not block the rest of the panel — a QA/test concern, not a UI surface.

**Frozen (D-11):** at timer zero the inherited overlay locks the panel and every control is disabled; the partial CSS/JS work stays "tal com està".

| Empty state body | JS: the single starting empty rule row **is** the scaffolding (no separate empty-state copy) |
| Error state | **None by design** — an already-used `(event, origen)` option is **disabled** (greyed, D-15), a composite action **disables** the destí select (D-17), and the row limit **disables** "Afegir JavaScript" (D-12). Additional fixed copy:
- **CSS forat labels are real CSS property names**, read-only monospace: `background-color:`, `border-color:`, `border-width:`, `top:`, `left:` / `right:`, `width:`, `height:`, `border-radius:` — plus the selector braces `.antena {` / `}`. Derived from the target CSS, never editable (GAME-06).

**Destructive-action inventory (Phase 3):** none. Removing a JS rule row is non-destructive undo; resetting a CSS forat (if provided) is likewise non-destructive. Freeze-at-zero is inherited and needs no new copy.
