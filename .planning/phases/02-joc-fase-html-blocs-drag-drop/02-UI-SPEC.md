# Phase 2 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope of this contract:** the **playable HTML phase** rendered inside the Phase-1 split shell (`renderActiveSplitScreen()` in `src/client/client.js`). It specifies the visual and interaction contract for the **drawer** (calaix — piece tray), the **nested slot board** (tauler), the **piece chips** (real tag+class labels, D-12), the **magnet+rebound snap** feel (D-09), the **distractor rebound** read (D-11), the **own progress N/8** indicator (D-14), the **initial hint** (D-14), and the **live preview** framing (D-13). It ALSO fixes the Admin team-card "N/8 peces" progress fill reserved by Phase 1 (D-15, D-08 of Phase 1).

**Inheritance:** this contract EXTENDS Phase 1's UI-SPEC. All Phase 1 tokens, the 60/30/10 split, the phase color-coding (`--phase-html`), typography, spacing, and the frozen overlay are reused verbatim. Only NEW visual/interaction surfaces introduced by the HTML game are specified here. Executor MUST reference existing tokens in `src/client/shared/tokens.css`, never hard-coded literals.

**Audience driver (rectora):** adults ~40 anys, zero prior experience, ~10 min prior talk. Every discretionary choice resolves toward minimal cognitive load, icons/glyphs over text, and no-frustration (UX-01). Wrong actions are never punished with error text — they are answered by motion (magnet rejection + gentle rebound).

## Spacing Scale

Reuses Phase 1 `--space-*` scale unchanged (4 / 8 / 16 / 24 / 32 / 48 / 64). No new spacing tokens.

| Token | Value | Phase-2 usage |
|-------|-------|---------------|
| xs | 4px | Icon/glyph-to-label gap inside a piece chip; chip internal padding |
| sm | 8px | Gap between piece chips in the drawer; slot inner padding |
| md | 16px | Gap between the drawer, board, progress and hint blocks; board container padding |
| lg | 24px | Outer padding of nested container frames (section/cap) |
| xl | 32px | Action-panel outer padding (inherited from `.action-panel`) |

Exceptions:
- **Hit target (reuses `--hit-target-min: 44px`):** every draggable **piece chip** and every **drop slot** renders at a minimum 44px hit area, even for small elements (`ull`, `nas`), to forgive imprecise dragging under time pressure. This pairs with SortableJS `emptyInsertThreshold: 40` (the magnet radius, D-09) — the visual hit area and the magnet radius are intentionally aligned so "near enough" visually equals "near enough" mechanically.
- **Slot board layout:** the nested container frames use flex/grid to loosely mirror the robot's spatial layout (see Layout Contract), not the spacing scale, for their relative positioning; the scale still governs padding inside each frame and slot.

## Typography

Reuses Phase 1 roles (Body 16/400/1.5, Label 14/600/1.4, Heading 24/600/1.2, Display 56/600/1.0) unchanged. Exactly 4 sizes, 2 weights — no new sizes or weights.

**New token — monospace family for code labels:**

| Token | Value | Rationale |
|-------|-------|-----------|
| `--font-family-mono` | `ui-monospace, SFMono-Regular, Menlo, Consolas, monospace` | Tag+class labels (D-12) render in monospace to signal "això és HTML de veritat" without inventing parallel vocabulary. System stack = zero bundle cost, no new asset. |

| Role | Size | Weight | Line Height | Phase-2 usage |
|------|------|--------|-------------|---------------|
| Body | 16px | 400 | 1.5 | Initial-hint micro-copy |
| Label | 14px | 600 | 1.4 | Progress "N/8 peces"; nested container frame labels |
| Code label | 14px | 400 | 1.4 | Tag+class label on piece chips and empty slots — **monospace**, 400 weight (code reads lighter than UI labels), never editable |
| Heading | 24px | 600 | 1.2 | (inherited — team name in panel header) |
| Display | 56px | 600 | 1.0 | (inherited — timer) |

Notes:
- **Code labels ARE the literal HTML tag, with angle brackets.** Show the real tag — `<img class="antena">`, `<span class="ull">`, `<button id="nas">`, `<output id="boca">`, and the container frames `<section id="robot-contenidor">`, `<div id="robot-cap">`, `<div class="contenidor-ulls">`. Rendered as read-only monospace **text** (plain characters, never interpreted as markup — no `innerHTML`, no editable field; GAME-06 preserved). **Override (2026-07-03, Plan 02-03 checkpoint):** this REVERSES the earlier "NO angle brackets / bare name" UI-discretion resolution of the D-12 dilemma. The user explicitly requested the real HTML tags "tal i com et vaig passar el esquema html" (D-01) at the human-verify checkpoint; the bracketed literal is now the chosen form, aligning the labels with the actual schema instead of a simplified alias.
- Code labels are **read-only, never editable** — no `<input>`, no `contenteditable` (GAME-06). Rendered as plain text nodes.

## Motion Contract

New for this phase (snap feel is core to GAME-03 / D-09 / D-11). All motion is **decorative reinforcement** — the game is fully usable with motion disabled. New tokens declared in `tokens.css`:

| Token | Value | Usage |
|-------|-------|-------|
| `--motion-snap` | `150ms` | SortableJS `animation` value — magnet-in glide and rebound-return duration (D-09 "torna suaument") |
| `--motion-ease` | `cubic-bezier(0.2, 0, 0.2, 1)` | Standard ease-out for snap/return |

Behaviors:

| Interaction | Read | Spec |
|-------------|------|------|
| **Magnet-in (valid, D-09)** | Piece glides the last gap into the slot | SortableJS `animation: 150` + `emptyInsertThreshold: 40`. On drop, slot plays an **accept pulse**: scale `1 → 1.04 → 1` over ~200ms + a one-shot `--phase-html` ring flash |
| **Rebound (wrong type / full slot, D-09)** | Piece returns gently to the drawer | SortableJS native revert (~150ms `--motion-ease`). No message, no color |
| **Distractor rebound (D-11)** | "Això no hi va" — playful, not punishing | Same native revert PLUS a short **shake** on the returned chip: `translateX ±6px`, 3 oscillations, ~300ms, then rest. No error text, no red. The comic glyph (🍌/🛞/👟) already carries the "wrong" meaning; the shake confirms it |
| **Remove piece (D-10)** | Undo is easy and unpunished | Piece animates back to the drawer with the same 150ms glide. N/8 decrements. No confirmation dialog |
| **Initial hint (D-14)** | "Drag pieces into the holes" | An arrow (Lucide `MoveRight`/`ArrowRight`) bridging drawer→board, gently animated (loop translate ±4px). Fades out **permanently** the moment placement count first goes 0→1 |

**Reduced motion:** wrap the accept-pulse, distractor-shake, and hint-arrow loop in `@media (prefers-reduced-motion: no-preference)` (same pattern as the existing `.timer-display` critical pulse). With reduced motion: pieces still snap/return **instantly** (functionality preserved), the accept pulse becomes a static ring, the shake is omitted, and the hint arrow is shown static (no loop).

## Layout Contract

Phase-2-specific structural rules, inside the inherited `.active-split` (left `.action-panel` ~40%, right `.preview-frame` ~60%).

**Left action panel (~40%), top → bottom:**
1. **Panel header** (inherited): HTML phase badge (orange) + team name (Heading) + synchronized timer (Display).
2. **Progress N/8** — compact, glanceable, directly under the header.
3. **Drawer (calaix)** — horizontal wrap tray of piece chips (8 good + 2–3 distractors, shuffled).
4. **Initial hint arrow** — sits between drawer and board, pointing from tray toward the slots; dismisses on first placement.
5. **Slot board (tauler)** — the nested container frames. Its visual arrangement **loosely mirrors the robot's spatial layout** so the left board and the right preview correspond (aids the structure↔reward double reading):
   - `section#robot-contenidor` frame contains everything.
   - Two `antena` slots across the top; two `orella` slots flanking the sides.
   - `div#robot-cap` frame in the middle, containing: `div.contenidor-ulls` frame (two `ull` slots side by side), then `nas` slot, then `boca` slot stacked.

**Right preview (~60%):** the real rendered robot (D-13), inherited iframe. Antennae/ears (images) and "BEEP BEEP" appear immediately; empty `span.ull` / `button#nas` are invisible until the CSS phase — this asymmetry is intentionally covered by the always-visible left board. **No target thumbnail** (D-14 — the teacher projects the Plana Model).

**Frozen (D-11):** at timer zero the inherited overlay locks the panel AND all SortableJS instances are disabled; the partial robot stays "tal com està" and travels to Phase 3.
