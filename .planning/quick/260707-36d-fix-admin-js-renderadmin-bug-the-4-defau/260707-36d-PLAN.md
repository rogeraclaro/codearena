<objective>
Fix a first-render bug in `renderAdmin()` (src/client/admin.js) where the 4 default team names that `buildRegistrationBlock()` prefills into `#team-names-input` get overwritten by an empty string.

Root cause: `preservedValue` is captured from `existingInput`, which is `null` on the very first render (before any textarea exists), yielding `preservedValue = ''`. The unconditional restore `newInput.value = preservedValue` then clobbers the freshly-prefilled defaults with `''`.

Fix: only run the restore step when `existingInput` was non-null, so the CR-02 preserve-in-progress-typing behavior survives for later re-renders but never runs on the initial render.

Purpose: Teacher sees the 4 pre-filled default team names when the Admin panel first loads, instead of an empty textarea showing only placeholder text.
Output: One-line guard change in `renderAdmin()`.
</objective>
